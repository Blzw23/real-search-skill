# 质量门禁

完成前先验证，再声明完成。

## 资料完整性

- 有 `研究计划.md`。
- 有 `资料索引.md`。
- 有阶段日志或等价进度记录。
- 正式结论能追溯到来源。
- 区分了官方文档、源码、论文、二手资料。

## 源码证据

如果 clone 了仓库，必须记录：

- 仓库 URL。
- commit hash。
- clone 策略，例如 shallow clone 或 sparse checkout。
- 阅读过的关键文件路径。
- 哪些部分没有读。

## 论文证据

如果引用论文，必须记录：

- 标题。
- 链接。
- 年份。
- 解决的问题。
- 方法结构。
- 对当前目标的启发和局限。

## 判断质量

检查最终报告是否回答：

- 主流成熟方案是什么？
- 前沿方法是什么？
- 哪些只是研究原型？
- 哪些适合用户当前目标？
- 有无类似现成项目？
- 如果要自研，差异化在哪里？
- 下一步如何落地？

## 风险标注

以下内容必须标注：

- 没有读源码，只读文档。
- 网络无法访问或 clone 失败。
- 只看到旧版本资料。
- 判断来自二手文章。
- 结论依赖用户未确认的假设。

## 外部副作用

默认不得执行：

- `git push`
- 发布到网站/社媒/公众号
- 发送邮件或消息
- 修改远端资源
- 调用会写入外部系统的 API

如果用户另行要求这些动作，把它们当成独立任务处理，不属于本 skill 的默认研究流程。
