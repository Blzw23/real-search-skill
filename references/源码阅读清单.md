# 源码阅读清单

源码阅读要服务于理解设计，而不是机械列目录。

## 入口层

- README、quickstart、examples 是否揭示主要使用方式。
- CLI 入口或 SDK 入口在哪里。
- 用户请求如何进入 runtime。

## 核心抽象

重点找：

- Agent / Runner / Runtime
- Tool / Skill / Plugin / MCP
- Message / State / Context
- Memory / Store / Checkpoint
- Trace / Event / Log
- Permission / Sandbox / Policy
- Planner / Worker / Reviewer / Multi-agent orchestration

记录抽象之间的关系，而不是只摘类名。

## 运行循环

追踪一次典型调用：

```text
input -> plan/think -> tool selection -> tool execution -> observation -> state update -> reflection/retry -> final
```

如果项目不是显式 agent loop，也要说明它如何实现类似效果。

## 扩展机制

检查：

- 如何注册工具。
- 是否支持 schema。
- 是否支持动态发现。
- 是否支持 MCP 或外部进程。
- 是否有权限确认。
- 是否能组合多个 skill/tool。

## 可靠性

检查：

- 错误分类。
- retry/backoff。
- timeout/cancellation。
- structured log。
- trace/span/event。
- eval/test harness。
- dry-run 或模拟模式。

## 安全边界

检查：

- 文件系统权限。
- shell/浏览器权限。
- 网络访问。
- secret 管理。
- sandbox/container。
- 人类确认机制。

## 记录格式

```markdown

| 路径 | 作用 | 读到的设计 | 证据强度 |
| --- | --- | --- | --- |

## 调用链

1. ...
2. ...

## 判断

- 可借鉴：
- 不适合照搬：
- 待验证：
```

## 什么时候停止深挖

满足以下条件即可阶段性停止：

- 能说清核心抽象和主流程。
- 能解释它如何扩展工具/记忆/权限/trace。
- 能指出 3-5 个可借鉴点和 2-3 个不适合点。
- 已记录未读区域和后续阅读建议。
